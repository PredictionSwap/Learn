# On Exposure

## Introduction

In prediction markets the term **exposure** is commonly used to describe how the value of an account depends on the outcome of an uncertain event.

In PredictionSwap this concept appears in two closely related forms. Because of this, the word **exposure** may refer to more than one object depending on context.

This document clarifies the formal representations used within the protocol.

PredictionSwap represents the economic state of an account using **outcome-indexed value vectors**. These vectors describe the value of an account under each possible outcome of a market.

Two related vectors are used:

- **Account Value**
- **Position Value**

These are connected through a simple decomposition of account value into **freecash** and **position value**.

---

## Outcome Space

Consider a market with outcome set

```
Ω = {ω₁, ω₂, …, ωₙ}
```

Exactly one of these outcomes will occur when the market resolves.

PredictionSwap represents account state by specifying the value of the account under each outcome in this space.

---

## Account Value

The **Account Value Vector** represents the value of an account conditional on each possible outcome.

```
e = (e₁, e₂, …, eₙ)
```

where

```
eₖ = value of the account if outcome ωₖ occurs
```

The Account Value Vector fully specifies the economic state of an account within the protocol.

At resolution, the realized outcome selects the corresponding component of this vector to determine the account’s final value.

---

## Position Value

The **Position Value Vector** represents the portion of account value generated by market positions.

```
r = (r₁, r₂, …, rₙ)
```

where

```
rₖ = value contributed by positions if outcome ωₖ occurs
```

Position Value therefore captures the **outcome-dependent component** of account value.

---

## Freecash

In addition to position value, accounts may hold **freecash**.

Freecash is outcome-independent capital which contributes equally to the value of the account under every outcome.

If an account holds freecash amount

```
c
```

then this contributes

```
(c, c, …, c)
```

to the **Account Value Vector**.

---

## Decomposition of Account Value

Account Value can therefore be decomposed into **freecash** and **position value**.

```
Account Value = Freecash + Position Value
```

or in vector form

```
e = r + c
```

where freecash is understood as a constant vector across all outcomes.

---

## Exposure

In informal discussion the term **exposure** refers to the way an account’s value varies across outcomes.

Depending on context this may refer to either:

- the **Account Value Vector**, which describes total outcome-contingent value, or  
- the **Position Value Vector**, which describes the outcome-contingent value generated by positions.

The distinction between these two representations is the **inclusion or exclusion of freecash**.

---

## Summary

PredictionSwap represents account state using **outcome-indexed value vectors**.

The key objects are:

| Symbol | Name | Description |
|------|------|-------------|
| e | Account Value | value of the account under each outcome |
| r | Position Value | value contributed by positions |
| c | Freecash | outcome-independent capital |

These are related by

```
e = r + c
```

This representation separates **outcome-dependent value** from **outcome-independent capital** while preserving a complete description of account state.
